# Projects Page Contents Visual Tweak Design

## Goal

Make the Projects page contents and first project section feel better balanced.

## Design

The contents navigation should stay directly below the `Projects` heading, but its box should shrink to the width of the contents instead of spanning the full page column. It should remain left-aligned and vertical, with enough maximum width for future project names.

The Chatham House collaboration panel should sit slightly lower in the Scenario Builder intro row so it reads as supporting context for the project, not as a page-level panel floating above the project copy.

Rename the contents entry from `Chatham House future worlds` to `Scenario Builder`, and capitalize the first project heading as `Scenario Builder`.

## Testing

Update the Projects page test to assert that the contents link is named `Scenario Builder` and the first project heading uses the same capitalization.
